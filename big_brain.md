# big_brain 功能说明

## 1. 项目定位

`big_brain` 是“大小脑协同”系统中的大脑部分，负责把自然语言任务转成机器人可执行的动作序列，并结合历史经验、语义地图和规则判断完成任务规划。

从当前代码来看，这一模块主要承担 4 件事：

1. 接收用户任务并生成执行计划
2. 从历史任务中检索相似案例，为当前规划提供参考
3. 将自然语言中的目标物体解析为语义地图中的具体对象
4. 调用底层动作 API 执行导航、抓取、放置等原子动作

## 2. 整体执行流程

入口文件是 `big_brain.py`，核心类为 `BigBrain`。当前主流程如下：

1. 读取历史任务 `memory/rag_history.json`
2. 初始化 `RAGManager`，加载句向量模型
3. 初始化 `PlannerLLM`
4. 接收任务指令
5. 用 RAG 检索历史相似任务
6. 将基础 prompt、历史参考任务和当前任务拼接成最终 prompt
7. 调用大模型生成 Python 代码形式的执行计划
8. 读取 L2 语义地图对象集 `objects`
9. 将动作函数、对象字典和运行上下文注入执行环境
10. 使用 `exec()` 执行规划结果
11. 任务结束后将结果整理为历史记录

可以把它理解为：

自然语言任务 -> 检索历史经验 -> LLM 生成计划代码 -> 执行动作 API -> 判断结果

## 3. 当前支持的核心功能

### 3.1 自然语言任务规划

`model/llm.py` 中的 `PlannerLLM.generate_code()` 负责调用大模型，把用户输入转成一段可执行 Python 代码。

当前设计特点：

- 输出不是结构化 JSON，而是直接生成 API 调用代码
- 生成的代码允许使用循环、变量、中间计算
- prompt 中给了多个示例，帮助模型学习任务拆解方式
- 支持把复杂任务拆成多个原子动作串联执行

示例能力包括：

- 移动到某个坐标或某个物体附近
- 拾取指定物体
- 放置到指定位置或相对位置
- 根据两个物体的中间位置进行放置
- 进行重复或路径型运动

### 3.2 历史任务检索增强（RAG）

`model/rag.py` 中的 `RAGManager` 负责检索历史相似任务。

功能特点：

- 使用 `sentence-transformers` 句向量模型编码历史任务
- 通过余弦相似度选取最相近的历史命令
- 当相似度高于阈值时，把历史任务及其动作序列拼接进 prompt
- 帮助大模型参考已有成功范式，提高规划稳定性

当前数据来源：

- 历史数据文件：`memory/rag_history.json`
- 相似度阈值：`config.py` 中的 `RAG_SIMILARITY_THRESHOLD`

### 3.3 语义对象解析

`utils/utils.py` 中的 `parse_obj_name()` 用来把自然语言中的“目标物体描述”映射到语义地图对象。

它的工作方式是：

1. 将当前 `objects` 字典转成 prompt 输入
2. 结合 `prompt/object_prompt.py` 中的示例
3. 调用大模型生成一段筛选代码
4. 执行代码并读取 `ret_val`
5. 返回匹配到的对象名或对象列表

从示例看，当前希望支持的对象解析能力包括：

- 直接按名称查找，如 `desk`、`banana`
- 按类别筛选，如可吃的物体
- 按颜色筛选，如蓝色瓶子
- 按空间关系筛选，如最左侧的椅子
- 按相对关系筛选，如在某物体后面的物体
- 按场景语义筛选，如“有两个瓶子的桌子”
- 按机器人位置筛选，如离机器人最近的水果

### 3.4 原子动作执行

`action/robot_api.py` 定义了大脑层暴露给 LLM 的基础动作原语。

当前主要动作包括：

- `move_to_xy(x, y)`：移动到绝对坐标
- `move_to_obj_by_offset(Object, dx, dy)`：移动到目标物体附近的相对位置
- `pick_up_xy(x, y)`：在指定坐标拾取
- `pick_up_xy_arm(x, y)`：通过机械臂尝试抓取指定坐标
- `pick_up_obj(item)`：抓取指定物体
- `put_down_xy(x, y)`：在指定坐标放下
- `put_down_obj_by_offset(target, dx, dy)`：在目标物体相对位置放下

这些函数是规划代码中的“最小动作单元”，也是整个大脑执行层的核心接口。

### 3.5 机械臂控制接入

机械臂相关代码主要在：

- `action/robot_api.py`
- `action/arm.py`
- `action/RoArm-M3_Python/`
- `roarm_m3_zh.md`

当前已接入 `roarm-sdk`，并预留了通过 `RoArm-M3` 机械臂执行抓取姿态控制的逻辑。`pick_up_xy_arm()` 中已经包含：

- 根据机器人本体与夹爪位置估算可达性
- 通过 HTTP 连接机械臂
- 调用 `pose_ctrl()` 执行机械臂位姿控制
- 执行后回到初始位姿

这说明项目不只是高层规划，也已经开始和真实机械臂控制对接。

### 3.6 任务结果判断与纠错框架

`model/llm.py` 中的 `JudgeLLM` 负责动作后的结果判断与微调恢复。

框架上已经设计了三层能力：

1. 规则判断
2. 视觉判断
3. 微调重规划

具体包括：

- 根据动作类型解析目标参数
- 读取机器人位置、朝向、夹持状态
- 用规则阈值判断是否成功
- 需要时调用视觉模型查看动作后图像
- 让 VLM 返回结构化 JSON 判断结果
- 根据失败原因，生成恢复代码进行局部重规划

当前支持判断的动作类型编号：

- `1`：移动到坐标
- `2`：移动到物体相对位置
- `3`：在坐标抓取
- `4`：抓取指定物体
- `5`：在坐标放置
- `6`：在物体相对位置放置

### 3.7 视觉语义校验框架

`prompt/vlm_prompt.py` 和 `JudgeLLM.judge_VLM()` 共同定义了视觉校验链路。

视觉判断目标包括：

- 规则判断失败时，分析失败原因
- 规则判断成功时，检查语义上是否真的满足任务要求
- 识别抓空、抓错、目标被占用、放置关系错误等问题
- 输出可供恢复规划使用的语义级纠错建议

这一部分特别适合处理：

- “放在 A 和 B 中间”
- “放在桌子中央”
- “抓取蓝色杯子”
- “目标位置被别的物体占据”

## 4. 主要模块说明

### 4.1 `big_brain.py`

系统主入口，负责串起：

- 历史记忆加载
- RAG 检索
- Prompt 拼装
- 计划生成
- 代码执行
- 任务记录

### 4.2 `config.py`

集中管理系统配置，主要包括：

- LLM / VLM 的 API 配置
- 当前启用的模型
- RAG 模型与阈值
- 动作判断误差阈值
- 默认物体尺寸
- 目标检测物体类别列表

### 4.3 `model/`

模型相关能力目录：

- `llm.py`：任务规划、结果判断、重规划
- `rag.py`：历史任务检索
- `vlm.py`：当前为空，预留视觉模型封装位置

### 4.4 `prompt/`

保存各类 prompt 模板：

- `task_prompt.py`：任务规划 prompt
- `object_prompt.py`：对象解析 prompt
- `adjust_prompt.py`：失败后的恢复规划 prompt
- `vlm_prompt.py`：视觉检查 prompt
- `position_prompt.py`：位置相关 prompt 预留

### 4.5 `utils/`

主要提供执行辅助函数：

- 对象位置、尺寸、颜色读取接口
- 机器人位置、朝向、夹持状态读取接口
- 大模型调用封装
- 代码块提取
- L2 语义地图读取
- 图像转 base64

### 4.6 `memory/`

保存记忆数据，目前主要包含：

- `rag_history.json`：历史任务样本
- `l2.json`：语义地图数据预留

### 4.7 `test_api.py`

用于测试大模型 API 连通性，当前支持：

- SiliconFlow
- OpenAI 兼容接口

## 5. 从功能角度看，big_brain 已具备的能力

如果只从系统职责出发，`big_brain` 目前已经具备以下“功能模块”：

1. 任务理解：接收自然语言任务
2. 经验检索：从历史任务中找相似案例
3. 任务分解：把复杂任务拆成原子动作序列
4. 对象定位：把语言中的物体描述映射到对象集合
5. 动作编排：生成可执行 API 代码
6. 执行控制：调用导航/抓取/放置等动作接口
7. 状态判断：对动作结果进行规则级检查
8. 视觉校验：通过图像判断语义层面的执行正确性
9. 恢复规划：在失败后尝试生成修正动作
10. 经验积累：把任务保存为后续 RAG 的检索样本

## 6. 当前实现状态与限制

这部分很重要。当前项目已经把整体框架搭起来了，但有些能力还是占位或半完成状态。

### 6.1 已经比较明确落地的部分

- 大模型任务规划主链路已经打通
- RAG 检索框架已经可用
- 对象解析 prompt 机制已经具备
- 原子动作 API 已定义
- 机械臂 SDK 已接入
- VLM 判断与恢复规划框架已经设计完成

### 6.2 仍是 mock / 占位实现的部分

`utils/utils.py` 中多个接口目前返回固定值或简单占位值，例如：

- `get_obj_xy()`
- `get_obj_size()`
- `get_obj_rgb()`
- `get_robot_pos()`
- `get_robot_orientation()`
- `get_robot_arm()`
- `load_L2_memory()`
- `parse_obj_position()`

这意味着当前系统框架已经完整，但真正依赖实时地图、感知系统和机器人状态的部分还没有完全接实。

### 6.3 当前代码中需要注意的行为

- `big_brain.py` 里的任务指令目前是硬编码字符串，不是命令行交互输入
- `JudgeLLM.judge()` 开头直接 `return True`，因此当前判断与纠错逻辑实际上被短路，没有真正执行
- `_save_history()` 在主流程中被注释掉，因此新任务默认不会写回历史文件
- `HISTORY_PATH = "memory/rag_history.json"` 使用相对路径，通常需要在 `src/big_brain` 目录下运行

所以更准确地说，当前项目是“可运行的高层规划原型 + 真实硬件接入尝试 + 感知闭环预留框架”。

## 7. 适合展示的典型能力场景

根据现有 prompt 和示例，`big_brain` 适合展示以下任务：

- 导航到指定坐标
- 导航到某个物体附近
- 抓取桌面上的指定物体
- 把物体放到另一个物体旁边
- 把物体放到两个目标之间
- 围绕某个物体走矩形或圆形路径
- 重复执行一组动作
- 根据颜色、位置、类别筛选目标物体

## 8. 运行相关

当前目录下的 `readme.md` 给出的基本运行方式是：

```bash
pip install -r requirements.txt
python big_brain.py
```

依赖主要包括：

- `numpy`
- `sentence-transformers`
- `openai`
- `roarm-sdk`

如果要真正跑通完整链路，还需要准备：

- 可用的大模型 / 视觉模型 API
- RAG 向量模型路径
- 语义地图数据
- 机器人状态接口
- 机械臂或底层控制接口

## 9. 一句话总结

`big_brain` 的本质是一个“面向移动操作机器人的大脑原型系统”：它试图把自然语言任务转成机器人动作程序，并结合历史经验、语义对象解析、规则判断和视觉纠错，实现从任务理解到动作执行的高层智能闭环。
